How NED Freight works

Live carrier-rate calculator wired into Shopify checkout.

What it does

At checkout, NED Freight queries three carriers in real-time (Castle Parcels/Post Haste via GoSweetSpot, plus Mainfreight and Dailyfreight from rate cards), picks the cheapest, applies a markup, and returns a single shipping rate to Shopify.

The flow

Customer enters address at checkout ↓ Shopify POSTs cart + destination → /shopify/rates ↓ NED Freight builds packages from each item's weight (= CBM in kg) ↓ Query Castle Parcels (live) ─┐ Calculate Mainfreight (formula) ─┼─→ Pick cheapest Calculate Dailyfreight (formula) ─┘ ↓ Apply NED markup (10%) ↓ Return "Standard Delivery — $X" to Shopify ↓ Customer sees one clean shipping option

Pricing logic

Per-carrier raw cost (NED's actual cost to ship):

  • Castle Parcels / Post Haste — Live quote from GoSweetSpot API. Strip 8% GSS markup → true NED cost (incl. FAF + GST).
  • Mainfreight — MAX(min, base + per_m³ × cart_CBM) × FAF × GST
  • Dailyfreight — MAX(min, per_m³ × cart_CBM) × FAF × GST

Customer price = cheapest raw cost × 1.10 (NED markup).

Where CBM data lives

Every Shopify product variant has a weight field. We use weight = CBM (in kg-equivalent), so a 1m³ product has a weight of 1.0kg. Carriers price on cube anyway, so the conversion is transparent.

Sources, in order of authority:

  • Cin7 Option Cubic Meters (per-variant — most accurate)
  • Cin7 product-level Cubic Meters (fallback)
  • Parsed from product description (e.g. "120cm l x 38cm w x 45cm h")
  • Category default (e.g. all vases = 0.008m³)

Use the Product CBMs tab to edit any variant — changes sync to Shopify immediately.

Free shipping

Currently disabled for trial. Set FREE_SHIPPING_THRESHOLD env var to a real dollar value (e.g. 500) to enable free shipping over $500.

Configuration

Env vars (set in Render dashboard):

  • GSS_ACCESS_KEY, GSS_SITE_ID — GoSweetSpot API auth
  • SHOPIFY_STORE, SHOPIFY_ADMIN_TOKEN — for CBM admin UI
  • FAF_MULTIPLIER (default 1.30 = 30% fuel adjustment)
  • GST_MULTIPLIER (default 1.15 = 15% NZ GST)
  • NED_MARKUP (default 1.10 = 10% NED margin)
  • FREE_SHIPPING_THRESHOLD (default 999999 = effectively off)
  • ORIGIN_* — warehouse address (7 Paradyne Pl, Wigram, ChCh 8042)

Endpoints

  • POST /shopify/rates — Shopify carrier service callback (production)
  • POST /shopify/rates/debug — Same, but returns all carrier quotes for diagnosis
  • GET /api/quote?city=X&cbm=Y&qty=Z — Quick test endpoint
  • GET /api/cbm-list — All variants + current CBM
  • PUT /api/cbm — Update one variant's CBM
  • POST /api/reload-rates — Hot-reload carrier_rates.json

Maintenance tasks

When FAF or GST changes

Update FAF_MULTIPLIER in Render env vars and redeploy. No code change needed.

When Mainfreight/Dailyfreight rates change

Edit data/carrier_rates.json in the repo, commit + push. Or POST to /api/reload-rates after editing on the server.

When a customer disputes a freight charge

Use the Test a quote tab — enter their address and cart CBM, see all three carriers' quotes and which one was picked. Compare with their actual carrier invoice.

If GoSweetSpot returns odd cheap quotes

The endpoint has a safety filter: any quote below max($8, cart_cbm × $40/m³) is dropped. Adjust in live_rates.py if needed.

Known limitation: The cube method (deriving package dimensions from weight) assumes products are cube-shaped. Real boxes are rectangular, so for items priced on dimensional weight, our quotes can be ~5-10% higher than the absolute optimum. For pure volumetric pricing (all our carriers), it's exact.
